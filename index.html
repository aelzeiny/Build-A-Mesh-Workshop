Build-a-mesh Workshop

Pathfinding on abstract polygons

Here we will be showcasing a series of common level generation techniques used by major gaming studios!

Implementing pathfinding on grid-based maps is a fairly basic task. However, real-world spaces are hardly ever so orthogonal or straight-forward.

Implementing pathfinding on concave (abstract) polygons is a much more difficult task. This project will hopefully demystify some common practices that developers use in the industry.

Sound complicated? Don't worry. Here's a penguin.

STEP 3:

Drop the player on the map

Now that you've built a map, we have to configure the mesh for pathfinding. You migh think that every polygon vertex would be a pathfinding node, but we can optimize the number of nodes by only considering vertecies that are concave (innies) rather than convex (outties).

Click on the area in which the player will walk around in. This demonstration works best if you pick the biggest polygon

STEP 4:

Building the Mesh

Sit back and relax

Green polygon verticies are optimal for pathfinding

STEP 5:

A Fully Connected Graph

Using a line of site algorithm, every node is connected to every one of its neighbors. Move your mouse around every nook and cranny, and you'll notice that there are no blind spots in the mesh.

This means that you can get from any starting location to any ending location.
